Answers

What are the restrictions for Twebst Redistributable Kit?
  • You are not allowed to use Twebst Libray as a part of another library, recorder or a general purpose automation software product (like Twebst Automation Studio). Also you are not allowed to redistribute or sell Twebst Library as a standalone component.

    You are allowed to create software products that include Twebst Libray as long they add enough web automation business logic, specific to a concrete problem.

    See: Twebst Redistributable Kit License
Is it allowed to redistribute Twebst Automation Studio?
  • Redistribution is not permitted, Twebst Automation Studio is licensed per user.
Do I need to install Microsoft .NET Framework to run Twebst macros?
  • No, Twebst Library does NOT depend on any .NET Framework version but Twebst Web Recorder requires Microsoft .NET Framework 2.0. That means .NET Framework is only needed when creating macros and not when running them.
What are the system requirements for Twebst?
  • Twebst Automation Studio was tested against Windows 8, Windows Vista, Windows XP (32 and 64 bit) with Internet Explorer 6 - 10
Is Windows 64 bit supported?
  • Yes, Windows 64-bit is fully supported starting with Twebst Library 2.7 which can be used from 32-bit and 64-bit applications and it can also automate IE 64-bit.
Why does my script fail to start on Windows 64 bit with error: "Automation server can't create object"?
  • Prior to 2.7 version, Twebst Library was available only for 32 bit applications. That's why .js and .vbs Twebst web macros must be run by 32 bit versions of wscript.exe / cscript.exe which you may find in %windir%\SysWOW64 directory.
Can I use Twebst Library with Web Browser Control in C#/Windows Forms project?
  • Yes, Twebst Library can work with web browser control by attaching to ActiveXInstance underlying IWebBrwoser2 object (see AttachToNativeBrowser library method).
I come across following error message: "Can not properly create a new browser object. It is possible that TwebstBHO.dll is not properly registered". What should I do?
  • Reinstall the product
  • Manually register TwebstLib.dll and TwebstBHO.dll using regsvr32.exe tool (on Vista/Win7 Run as admin)
